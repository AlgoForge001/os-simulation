What's Actually Happening?
Boot sequence is the initialization process that transforms a powered-off machine into a running operating system. Each stage hands control to the next, checking hardware, loading firmware, then progressively loading the OS kernel and user environment. Modern UEFI boot is significantly faster than legacy BIOS because it can initialize hardware in parallel.
⚡ Real World
Linux systems using systemd can boot in under 2 seconds with an NVMe SSD. The original IBM PC (1981) took 30+ seconds just for POST. Android devices use a specialized init system optimized for embedded hardware constraints.

Key Differences
Multiprogramming: Multiple programs in memory. CPU switches only when a program does I/O (cooperative). Goal: maximize CPU utilization.

Multitasking: CPU switches rapidly between processes (preemptive, every ~20ms). Gives illusion of parallelism. What you use daily.

Multithreading: One process with multiple threads sharing code/data space. Threads are lighter than processes.

⚡ Real World
A Chrome tab is a process (for isolation/security). Each service worker inside it is a thread. Chrome has used this architecture since 2008, making it crash-resistant — one tab dying doesn't kill others.

What's Actually Happening?
When an application calls read(), the CPU executes a software interrupt (INT 0x80 on x86 or SYSCALL instruction). This flips the CPU privilege level from Ring 3 → Ring 0, the kernel validates arguments, performs the operation, then returns the result and switches back to Ring 3. This mode switch costs ~100-200 nanoseconds.
🔢 Fun Fact
Linux has ~450 system calls. Windows has ~1,800 (though many are undocumented). The most frequently called syscall in any program? Probably mmap() — memory mapping happens constantly. You can trace all syscalls a program makes with strace ./program.

🏆 Real World
The Linux kernel is technically monolithic but supports loadable kernel modules — making it practically modular. macOS uses a hybrid XNU kernel combining Mach (microkernel) + BSD (monolithic) components. This is why macOS can run Unix programs natively!

06 · OS Classification

Types of Operating Systems

Click cards to flip and see definitions, use cases, and real-world examples.

How to Choose an OS Type?
The choice depends on timing constraints, resource availability, and use case. A hospital life-support system needs a Hard Real-Time OS (guaranteed response in microseconds). Your laptop uses a Time-Sharing OS (best-effort, interactive). A scientific computing cluster uses a Distributed OS (coordinate thousands of nodes). An embedded sensor uses a tiny Embedded OS that fits in kilobytes.
🚀 Fun Fact
The Mars Curiosity Rover runs VxWorks, a Real-Time OS. When it took its first drive, VxWorks had to handle commands with a 14-minute communication delay from Earth. The entire rover navigation runs autonomously with real-time safety constraints.

07 · OS Services

Operating System Services

Services provided to programs and users — the OS's public-facing API.

Services vs System Calls
Services are high-level capabilities the OS provides (like "program execution" or "I/O operations"). System calls are the low-level mechanism used to request those services. Think of services as the what and system calls as the how. One service might require multiple system calls to implement.
📖 Exam Tip
OS services are often grouped into two categories: services for users (UI, program execution, I/O, file system, communication, error detection) and services for efficient system operation (resource allocation, accounting, protection, security).
